Игра "Ним" и другие игры с камешками (или конфетками - кому как больше нравится)

Игра 1. Имеется кучка из N конфет. Два игрока по очереди берут из неё конфеты, причём за один ход разрешается взять от 1 до М конфет. Проигрывает тот, кто не может сделать ход.

A. Напишите функцию, которая будет
- получать на вход числа М и N (где М - максимальное число предметов, которое можно брать, а N - текущее число предметов в кучке),
- спрашивать игрока, сколько предметов он хочет взять,
- проверять, можно ли взять столько предметов, переспрашивать, если нельзя,
- давать в ответе число взятых предметов.

B. Напишите функцию, которая позволит программе делать правильный ход. Она должна
- получать на вход числа М и N (где М - максимальное число предметов, которое можно брать, а N - текущее число предметов в кучке),
- вычислять, какое количество предметов надо взять, чтобы противник оказался в проигрывщной позиции,
- если невозможно поставить противника в проигрышную позицию, выбирать случайный ход из имеющихся возможных,
- давать в ответе число взятых компьютером предметов.

C. Напишите программу, которая выберет случайные числа N и М - такие, что число N зависит от М (допустим, оно больше, чем М*2, но не больше, чем М*5), а потом будет организовывать игру двух игроков (говорить, чей ход, сколько конфет осталось, кто выиграл, следить за правильностью ходов).

Может быть два варианта игры: 1) когда играют два человека, 2) когда человек играет с программой. Во втором случае человек выбирает, первым или вторым ему играть.

Как научить программу выигрывать?

Первый способ. Надо научиться выигрывать самому, для этого составить карту выигрышных и проигрыщных позиций. Например, если можно брать не больше 2 конфет, то позиция 0 будет проигрышной. Позиции 1 и 2 будут выигрышными, так как из них можно отправить противника в проигрышную позицию 0. А вот из позиции 3 любой ход приведёт в выигрышную позицию (1 или 2). То есть как бы мы ни пошли, у противника будет возможность выиграть. Позиция 3 - проигрышная. И так далее.

Если мы научим программу вычислять выигрышные и проигрышные позиции, ей остаётся каждый свой ход пытаться отправить противника в проигрышную позицию. А если такой возможности нет (то есть программа сама попала в проигрышную позицию), то случайным образом выбирать один из возможных ходов - авось противник запутается и ошибётся.

Второй способ. Пусть программа сама учится выигрывать. Напишем функцию, которая получает на вход текущую позицию, а в ответе даёт список (или кортеж) позиций, в которые из неё можно попасть, если играть по правилам. Программа должна сама составить карту выигрышных и проигрышных позиций, пользуясь этой функцией. Это может быть строка из символов "П" и "В" или список логических значений.

Программа перебирает все позиции, начиная с 0. Если из данной позици можно перейти в проигрышную, то позиция выигрышная. Если из данной позиции можно попасть только в выигрышные позиции, то позиция проигрышная.

D. Имеется кучка из N конфет. Два игрока по очереди берут из неё конфеты, причём за один ход разрешается взять не больше половины всех конфет. Проигрывает тот, кто не может сделать ход (это произойдёт, когда останется одна конфета). 1) Напишите программу, которая организует такую игру с двумя игроками. 2) Напишите программу, которая может сама играть за одного из игроков. За кого играть, выбирает человек. При этом программа играет либо оптимально, либо случайным образом, если оптимальные ходы отсутствуют.

В этой задаче можно самому составить карту выигрышных и проигрышных позиций и использовать её в программе.

E. Имеется кучка из N конфет. Два игрока по очереди берут из неё конфеты, причём за один ход разрешается взять М1, M2 или М3 (например, 3, 5 или 8) конфет. Числа М1, M2 и М3 выбираются случайным образом. 1) Напишите программу, которая организует такую игру с двумя игроками. 2) Напишите программу, которая может сама играть за одного из игроков. За кого играть, выбирает человек. При этом программа играет либо оптимально, либо случайным образом, если оптимальные ходы отсутствуют.

В этой задаче удобно составление карты выигрышных и проигрышных позиций переложить на программу.

F. Ним — математическая игра, в которой два игрока по очереди берут предметы, разложенные на несколько кучек. За один ход может быть взято любое количество предметов (большее нуля) из одной кучки. Выигрывает игрок, взявший последний предмет.

Напишите программу игры в Ним для трёх кучек. Количество предметов в каждой кучке задаётся в начале игры.

G. Напишите программу, которая умеет играть в эту игру и выигрывать, если есть такая возможность. Про выигрышную стратегию можно почитать здесь .

H. Напишите программу для игры в Ним-поддавки: игрок, взявший последний предмет, проигрывает. Про выигрышную стратегию можно почитать здесь.
